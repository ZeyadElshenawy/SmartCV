# "Responsible For" — Duty Language vs. Accomplishment Language

The most common weak-bullet pattern in real-world resumes is the "Responsible for" / "Tasked with" / "Duties included" opener. MIT CAPD: "While it might be technically accurate to say 'was responsible for delivering projects on time,' it's much more effective to say 'ensured projects were delivered on or ahead of schedule.'"

Duty language describes the role description, not what the person achieved. Candidates with identical duty bullets are indistinguishable; recruiters specifically scan for outcome-language candidates.

**Banned opener phrases:**

"Responsible for [X]" / "Tasked with [X]" / "In charge of [X]" / "Duties included [X]" / "Job involved [X]" / "Position required [X]" / "Functioned as [role]" / "Served as the [role]" / "Acted as [role]" / "Worked as the [role]".

**Why "Responsible for" is fatal.** It describes what you were *supposed* to do, not what you *did*. Anyone in your role is responsible for the same things. Duty language defeats the resume's purpose: differentiation from candidates with the same title.

**Rewrite mechanic.** Strip the duty opener. Find the strongest specific action you took. Lead with that action; follow with the measurable outcome.

**Worked examples:**

- BAD: "Responsible for managing the deployment pipeline."
- GOOD: "Maintained the GitHub Actions pipeline across 18 services; cut mean release time from 40 min to 6 min."

- BAD: "Tasked with onboarding new engineers."
- GOOD: "Onboarded 8 engineers across 3 cohorts; time-to-first-PR dropped from 9 to 3 days."

- BAD: "In charge of code reviews and QA."
- GOOD: "Reviewed 200+ PRs in Q3, catching 11 production-blocking bugs pre-merge."

- BAD: "Duties included docs, mentoring, code reviews."
- GOOD: split into separate accomplishment bullets, each with a metric.

- BAD: "Job involved working with cross-functional teams."
- GOOD: "Coordinated the iOS-to-Android port across 3 engineers and 1 designer; shipped on the original 8-week timeline."

**Edge case: scope-setter.** Sometimes "Responsible for" usefully signals scope. The acceptable scope-setting verb is **Owned** — conveys ownership without duty-language passivity. Example: "Owned the Authentication API (REST + JWT, 14 endpoints, 8K RPS peak) for 18 months".

**Compound failure: duty + vague.** "Responsible for working on multiple projects to improve efficiency across various stakeholder groups" — zero signal. Total rewrite from source CV.

**Validator rule.** Reject any bullet starting with the banned phrases. If >10% of resume bullets contain a duty opener, flag the whole resume for regeneration.

## Concrete rule for SmartCV

The bullet validator must reject any bullet starting with: "Responsible for", "Tasked with", "In charge of", "Duties included", "Job involved", "Position required", "Functioned as", "Served as the [role]", "Acted as [role]", "Worked as the [role]". Rewrite the bullet by leading with the strongest specific action verb and pairing it with the measurable outcome from the source CV. Use "Owned" as the scope-setter when the bullet legitimately needs to describe ownership of a scope (followed by specifics in subsequent bullets).
