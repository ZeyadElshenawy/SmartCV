# Design and Creative Action Verbs

Design bullets describe visual, UX, product, and architectural design work. Apply to UX/UI designers, product managers, frontend engineers describing UI work, and software architects describing system design.

**Visual / UX design:**
Conceptualized, Crafted, Created, Customized, Designed, Drafted, Illustrated, Mocked up, Prototyped, Rendered, Sketched, Storyboarded, Visualized, Wireframed.

**System / architecture design:**
Architected, Composed, Designed, Diagrammed, Engineered, Formulated, Modeled, Planned, Scoped, Specified, Structured.

**Iteration and improvement:**
Iterated, Polished, Redesigned, Refined, Restyled, Revised, Revitalized, Streamlined, Updated.

**Research-driven design:**
Interviewed (users), Mapped (journeys), Observed, Tested, Usability-tested, Validated.

**Verbs to avoid:**
- "Made" — too generic; use Built, Crafted, Designed.
- "Worked on" — say what you did (designed, prototyped, iterated).
- "Beautified" — informal and vague.
- "Did design for" — passive; use Designed.

**Pairing design verbs with outcomes.**
Design work is hardest to quantify, but the strongest bullets attach to research findings, usage metrics, or downstream impact:

- "Redesigned the checkout flow from 5 screens to 2; A/B test on 8,000 users showed cart-abandonment drop from 38% to 24%."
- "Wireframed a 12-screen mobile onboarding spec consumed by 3 engineers; shipped on schedule with 0 design-review rework cycles."
- "Designed a component-token system (colors, spacing, typography) adopted across 6 product surfaces, replacing 18 ad-hoc one-off styles."
- "Conducted 14 user interviews; the resulting persona map informed the navigation redesign that lifted day-1 activation by 9 points."

**Design-specific scale signals:**
- Surface count: "across N product surfaces", "for N customer-facing screens".
- Adoption: "used by N engineers", "consumed by N teams".
- Test outcome: conversion lift, task-completion rate, NPS shift, usability-test success rate.
- Research scope: "N user interviews", "N usability sessions", "N survey responses".

**Frontend-engineer-specific note.** Frontend engineers describing UI work should use a mix of design verbs and engineering verbs. Pure design verbs (Wireframed, Sketched) belong only when the engineer was actually owning the design phase — not just translating a designer's mockups to code. For "implemented a designer's spec", use Implemented or Built, not Designed.

**Product-manager-specific note.** PM bullets describing the product spec should use Defined, Scoped, Specified, Prioritized — not Designed (which implies hands-on craft). Reserve Designed for when the PM did the design work themselves.

## Concrete rule for SmartCV

Use design verbs (Designed, Wireframed, Prototyped, Architected) only when the source CV indicates the candidate owned the design phase, not just executed someone else's design. Always pair design verbs with either a research input (N interviews, N tests) or a downstream outcome (conversion lift, adoption count, surface count). For frontend engineers translating designer mockups to code, prefer Built or Implemented over Designed.
