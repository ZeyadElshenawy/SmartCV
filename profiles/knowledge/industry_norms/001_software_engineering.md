# Software Engineering — General Resume Conventions

A general software engineer resume (not specialized to frontend/backend/ML) follows a standard structure recognized across the industry: contact info, professional summary, skills, work experience, projects, education. Indeed's software-engineer-resume guide (2026) and MIT CAPD's resume guidance converge on this layout.

**Section order (most-recent-first within each section):**
1. Contact information (name, email, phone, location, LinkedIn, GitHub).
2. Professional summary (2–3 sentences, lead with role + years + one concrete proof point).
3. Skills (3–6 grouped sub-categories — see `ats_rules/012_skills_section_layout`).
4. Work experience (3–5 bullets per role for recent roles, 2–3 for older).
5. Projects (especially for engineers under 5 years; senior engineers may omit unless the projects are exceptional, e.g., open-source maintainership).
6. Education.
7. Certifications (if applicable: AWS, GCP, Kubernetes, security certs).

**Skills to highlight (per Indeed 2026 software-engineer guide):**
- Computer programming and data structures / algorithms.
- Programming languages (the actual languages you use, not every one you've ever touched).
- Software development life cycle.
- Database design and optimization.
- For senior roles: leadership, system design, cross-functional collaboration.

**Bullet conventions:**
- Name the language, framework, or tool. "Built a Django REST API serving 12K DAU" beats "built a web application".
- Quantify with engineering metrics: latency (ms), throughput (RPS), error rate (%), build time (min), test coverage (%), deploy frequency, uptime (%), incident count.
- For incident work, name the system and post-fix outcome.
- Include scale: users served, services owned, or PRs reviewed beat lines of code.

**Open-source contributions.** Include an "Open Source" section between Projects and Education for meaningful OSS contributions. For maintainers, include stars, weekly downloads, or contributor count.

**GitHub URL placement.** Engineering roles expect a GitHub link in the contact block. Recruiters click through to verify code quality. Omit the link if the profile is empty or only forks.

**Common mistakes:**
- 25-language Skills section. Cap at 12–15 hard skills, ordered by current proficiency.
- Tag-list bullets without saying what was built.
- Long company-description paragraphs. Recruiters know what Google does.
- University coursework after 2+ years of experience. Coursework is intern/new-grad only.

**Length:** 1 page for under 7 years, 2 pages above.

## Concrete rule for SmartCV

For a generic software-engineer role (no specialization extracted from the JD), generate the standard 6–7 section structure: Contact, Summary, Skills (grouped), Work Experience (3–5 bullets per recent role), Projects (for under-5-year candidates), Education, optional Certifications. Every Work Experience bullet must name a specific language/framework/tool AND a quantified or artifact-named outcome. Limit Skills section to 12–15 hard skills total.
