# QA and Test Engineering — Resume Conventions

QA / test engineering covers manual exploratory testing, automated tests, performance testing, and quality engineering. Wikipedia's software-testing article (2026) breaks the scope into testing levels (unit, integration, e2e), automation, code-coverage (function/statement/decision), and regression testing — "the largest test effort in commercial software development."

QA vs. SDET matters: QA implies manual exploratory plus some automation; SDET implies ownership of test infrastructure as code. Resumes should signal which.

**Tech-stack categories:**

- **Frameworks:** JUnit / TestNG (Java), Pytest (Python), Jest / Vitest (JS), RSpec (Ruby), Go testing, NUnit / xUnit (C#).
- **E2E / browser:** Selenium, Playwright (dominant), Cypress, WebdriverIO, Puppeteer.
- **Mobile:** Appium, Espresso (Android), XCUITest (iOS), Detox, Maestro.
- **API:** Postman, REST Assured, Karate, Pact (contract testing), Newman.
- **Performance / load:** JMeter, k6, Locust, Gatling, Artillery, Lighthouse.
- **Test management:** TestRail, Zephyr, Xray, qTest, Allure.
- **CI:** GitHub Actions, GitLab CI, CircleCI, Jenkins; flake detection (Datadog Test Visibility, BuildPulse, Trunk).
- **Coverage:** JaCoCo, coverage.py, Istanbul, Codecov.

**Metrics:**
- **Coverage:** statement / branch %, # cases, # services with CI suites.
- **Defects:** detection rate, escape rate, MTTD.
- **Flake:** flaky-test rate, false-positive rate.
- **Cycle:** suite duration, regression-cycle time.
- **Adoption:** # teams onboarded, # services migrated.

**Strong bullet examples for QA / test:**

- "Built and maintained the Playwright e2e suite for checkout (38 scenarios, 4 browsers × 3 viewports); flake dropped from 18% to 1.4% in 6 months; runtime fell from 47 min to 14 min."
- "Designed the Pact contract-testing layer between 12 microservices; caught 14 breaking changes pre-merge in Q3; API-related defect escape fell from 3/quarter to 0."
- "Authored 220+ Pytest cases for the pricing service (94% line, 86% branch coverage); QA caught 19 bugs/release (up from ~6) before merge; production incidents fell from 4/quarter to 1/quarter."
- "Owned the k6 + Grafana load-test framework for 8 top-traffic services; caught 3 capacity bottlenecks pre-launch in 2024."
- "Onboarded 6 teams to Playwright + Allure; e2e coverage on critical paths rose from <12% to 60%+; release sign-off fell from 3 days to under 1."

**QA-specific anti-patterns:**

- Listing every test tool. Pick 3–4 you've shipped work in.
- Bullets without defect or coverage metric. "Wrote 100 test cases" — what bugs caught, what coverage, what incidents prevented?
- "Performed manual testing" without exploratory-charter detail.
- Claiming SDET without ownership of test infrastructure code.

**Shift-left.** Modern quality engineering involves working with developers earlier (design / PR review). Mid+ candidates need one bullet on developer-practice influence: PR templates, pre-merge contract tests, post-mortem participation.

## Concrete rule for SmartCV

For QA / test roles, surface the stack across Test frameworks, E2E, Mobile, API, Performance, and CI sub-groups. Quantify bullets with coverage %, flake rate, defect-escape rate, and CI duration. Distinguish between manual / exploratory work, automated test authorship, and SDET / framework-ownership work. For mid+ candidates, generate at least one bullet showing influence on developer practices (shift-left).
