# Intern / Student Resume Conventions

Intern resumes are evaluated on potential, not track record. They demonstrate (a) technical fundamentals (coursework + projects), (b) execution evidence (shipped projects, hackathons, OSS), (c) conscientiousness signals (GPA if strong, awards). MIT CAPD: include class projects, competitions, and personal projects alongside formal work experience.

**Length:** strictly one page.

**Section order:**
1. Contact info.
2. Education (top — strongest credential at this stage).
3. Skills.
4. Projects (3–5 strongest, with links).
5. Work experience (internships, part-time, research assistantships).
6. Awards / honors (if relevant).

**Education specifics:**
- Degree, university, expected graduation (`Expected May 2026`).
- GPA only if 3.5+ on 4.0 scale (3.7+ for FAANG-tier). Include local-scale equivalent for non-US.
- Relevant coursework: 4–6 directly-relevant courses max. Skip generic intros.
- Honors: Dean's List (with semester count), scholarships.

**Projects specifics:**
- 3–5 projects with name, tech stack, 1–3 bullets, GitHub or live link.
- Prefer shipped projects (deployed, in app store, 10+ GitHub stars) over coursework.
- Hackathon projects fine if they shipped.

**Work experience without internships:**
- Part-time jobs (TA, tutor, retail) if they show responsibility or work ethic.
- Research assistantships count as work, not coursework.
- Drop high-school jobs after sophomore year.

**What NOT to include:**
- "Objective: To gain valuable experience..." — outdated.
- Generic computer-literacy skills (MS Office) — assumed.
- Photo, marital status, religion (for international apps). MENA differs — see `mena_context/`.

**Bullet pattern for intern projects:**
CAR / X-Y-Z still apply. With limited metrics, substitute:
- **Scope:** lines of code, files, modules, features.
- **Tech stack depth:** signals production-tier choices, not tutorial copy.
- **Process signals:** "wrote 28 pytest cases", "set up GitHub Actions CI", "deployed on Railway".
- **Recognition:** stars, hackathon placement, course awards.

**Strong intern bullet examples:**

- "Built a CLI tool converting Markdown to Anki flashcards (Python + Click); 1.2K LOC, 24 GitHub stars, used by 40+ in r/medicalschool."
- "Co-led a 4-person team in the university hackathon (24h); shipped a real-time campus-shuttle tracker (React Native + Firebase + GTFS); placed 2nd of 32."
- "Contributed 6 PRs to Django REST Framework docs; 4 merged including a throttle-class clarification referenced in the project newsletter."

## Concrete rule for SmartCV

For intern / student resumes, generate one-page output with Education at the top, followed by Skills, then Projects (3–5 with links), then Work Experience. Include GPA only if 3.5+. Include relevant coursework only if 4–6 directly-relevant courses can be named (skip generic intro courses). Generate Project bullets with scope and tech-stack depth as substitutes for missing outcome metrics. Never include "Objective", high-school jobs after sophomore year, or generic computer-literacy skills.
